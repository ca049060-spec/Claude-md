# Resonance Forecasting Boundary

## Plain statement

The Resonance Web is not a prophecy engine.

It becomes forecast-testable only when signals are registered before outcomes are known and then reviewed later against baselines.

## What the model can do now

- structure live signals
- freeze assumptions
- identify source quality
- mark unknown future-dependent features
- recommend bounded action
- prevent overreaction
- create reviewable records

## What the model cannot honestly claim yet

- predictive superiority
- statistical validity
- reliable causal forecast power
- correctness on unresolved public events
- superiority over simple baselines

## Valid evidence of improvement

The model improves only if reviewed prospective signals show it beat simple baselines such as:

1. always-watch
2. majority-class
3. source-quality-only
4. actionability-only
5. common-sense bounded-action rules

## Operating boundary

Every Resonance output must separate:

- fact
- interpretation
- assumption
- unknown
- action
- review trigger

## Stop condition

If future reviewed signals do not beat baselines, keep the Resonance Web as a teaching/cascade-analysis tool and stop calling it a predictor.
