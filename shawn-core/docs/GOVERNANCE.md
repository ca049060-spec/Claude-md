# Governance

## Promotion path
workflow → checklist → skill → plugin

A workflow becomes a skill only after:
1. repeated three times
2. produced real-world value once
3. has clear inputs/outputs
4. has a safety boundary
5. can be tested

A skill becomes a plugin only after:
1. used across at least two projects
2. has schemas/templates
3. has privacy review
4. has an owner and version
5. passes validation

## Anti-overbuilding rules
- No new dashboard until 10 ledger items and 3 completions.
- No new connector until a current skill needs it.
- No new agent unless an existing lens failed.
- Archive ceremony when action is pending.
