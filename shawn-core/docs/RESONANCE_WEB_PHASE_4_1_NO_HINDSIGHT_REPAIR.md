# Resonance Web Phase 4.1 — No-Hindsight Repair

## Status

Phase 4.1 exists to repair the main weakness exposed in Phase 4: historical backtests were hindsight-contaminated.

This phase separates the Resonance Web into two modes.

## Mode A — Cascade teaching mode

Use this for historical events after the outcome is known.

Allowed:
- map cascades
- study feedback loops
- learn distortion patterns
- compare domains

Not allowed:
- claim forecast accuracy
- tune old cases to fake predictive power
- hide hindsight scoring

## Mode B — Prospective signal mode

Use this for live signals.

Rules:
- register the signal before outcome is known
- freeze information cutoff date
- store dated public sources available at cutoff
- score only knowable features
- mark future-dependent features as unknown
- save prediction label, confidence, bounded action, and review date
- review only after the signal matures

## What this fixes

- reduces hindsight contamination
- creates a live prediction ledger
- prevents fake 100% backtests
- separates teaching from testing
- forces unknown future-dependent features to stay unknown

## What remains unproven

The model is not yet a proven predictor. It is a structured signal registry and decision-support framework until prospective reviews mature.
