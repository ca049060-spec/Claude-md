# skill-harvester

## Purpose
Find repeated workflows worth turning into skills.

## Rules
- Do not create new skills automatically.
- Propose at most 5.
- Rank by real-world value.
- Reject anything that is only clever architecture.

## For each proposal include
1. skill name
2. when to use it
3. files it would read
4. tools/connectors needed
5. risk level
6. scope: personal, project, or plugin
7. first test case
8. build / wait / delete recommendation

## Promotion rule
A workflow becomes a skill only after:
- it repeats 3 times
- it produces real-world value once
- it has a clear safety boundary
