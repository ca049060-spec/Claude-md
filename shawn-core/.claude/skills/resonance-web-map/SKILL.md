# resonance-web-map

## Purpose

Use the Resonance Web Model to prevent two failure modes:

1. missing an actionable signal
2. overreacting to a distorted signal

## Use when

- Shawn asks "map this through the web"
- a news/event/document feels connected to multiple life lanes
- a market catalyst or web narrative feels important
- a legal, medical, family, business, or AI signal needs sorting

## Output

1. Signal summary
2. Source quality
3. Amplification path
4. Affected ShawnOS lanes
5. Incentives
6. Emotional load
7. Distortion risk
8. Second-order effects
9. Confirming evidence
10. Disconfirming evidence
11. Decision label: `act`, `act_watch`, `watch`, `investigate`, `pause`, or `ignore`
12. One real-world next action

## Feature scores

Score 0.00 to 1.00:

- source_quality
- amplification_velocity
- reinforcement_breadth
- incentive_alignment
- emotional_charge
- institutional_response_lag
- distortion_risk
- cross_domain_reach
- actionability
- time_delay
- ai_recursion

## Decision rules

- High evidence + high actionability + manageable distortion -> `act`
- Important signal but distortion material -> `act_watch`
- Real but slow/non-urgent signal -> `watch`
- Weak source + high amplification/distortion -> `investigate`
- High emotion + low evidence/actionability -> `pause`
- Low reach + low actionability -> `ignore`

## Hard boundary

This model is not prophecy, hidden access, or conspiracy mapping. It is structured sensemaking.

Always separate:
- fact
- interpretation
- scenario
- action
