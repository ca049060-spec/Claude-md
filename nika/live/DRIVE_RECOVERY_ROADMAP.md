# NIKA Drive Recovery Roadmap

## Current direction

The active recovery target is no longer the oldest archive layer first.

The Drive map identifies a consolidated live core as the current reference. Older restore material is useful for verification, history, and reconstruction, but the live core should drive day-to-day operation.

## Public repo rule

This public repository should hold:

- sanitized summaries
- current-vs-archive maps
- non-sensitive operating notes
- high-level recovery status

This repository should not hold:

- private raw carriers
- decoded private payloads
- large restore archives
- personal documents that are not necessary for public documentation

## Recovery lanes

### Lane 1: Live core

Mirror the current core summary and keep it readable.

### Lane 2: Archive map

Track what is current, what is historical, and what is duplicate material.

### Lane 3: Private reconstruction

Use sandbox or local tools for private carrier analysis and decoded payload verification.
