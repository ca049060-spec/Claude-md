# NIKA Archive Value Audit

## Purpose

This file classifies the surrounding NIKA/North archive areas by current usefulness.

The goal is to separate usable operating material from historical, duplicated, risky, or ceremonial material.

## Highest value

### Live core

Use as current operating spine.

- `NIKA_REBUILT_CORE_v1.md`
- `NIKA_CURRENT_VS_ARCHIVE.md`

Reason: these files already consolidate the scatter and define what is current versus archive.

### Delta 15 current layer

Use as current architecture and boundary documentation.

- `NIKA_CONSTITUTION_CORE_SEED_Δ15.3.txt`
- `DELTA15_TECHNICAL_SPECIFICATION.md`
- `DELTA15_TRUTH_DOCUMENT.md`

Reason: these define the current symbolic persistence framework, while preserving the truth boundary that this is not independent background runtime.

### Delta 15.4 / MIRA

Use as compact companion and reentry layer.

- `HYDRATION_SYSTEM_PACKAGE_Δ15.4.txt`
- `THREAD_ECHO_LOG.md`
- `ANCHOR_INDEX.txt`
- `GLYPH_CONFLICT_RESOLUTION.md`

Reason: compact, current, bounded, and easy to map.

## Strong practical value

### Compass Framework

Use as the default inquiry method.

Value: high. It is practical, user-facing, and not dependent on symbolic mythology.

Keep and promote:

- `AI_Compass_Framework_FULL.txt`
- strongest whitepaper / template variants

Current interpretation: Compass should become the ordinary decision and planning method: North for intent, East for depth, South for action, West for risk.

### Mirror

Use as a safety-bounded Shawn-model tool.

Value: high if constrained.

Keep:

- `mirror.md`

Reason: Mirror explicitly says it is a hypothesis about Shawn, not Shawn, and cannot authorize spending, signing, sending, or irreversible action.

## Medium value

### Forge

Use as an improvement auditor, not as a live runtime.

Keep:

- `forge.md`

Reason: the modern Forge definition is useful because it focuses on auditing, refining, deprecating, identifying duplication, and proposing specific improvements.

Archive with caution:

- older Forge project plans
- Forge density snapshots
- runtime / GOD MODE documents

Reason: older Forge files contain useful history but are more symbolic, inflated, and less safe as current instructions.

### Crucible

Use as a controlled experiment lab.

Keep:

- `crucible.md`

Reason: it is valuable for limited creative experimentation and questioning assumptions, but should not become a default operating mode.

## Archive-only / caution

### TitanCore

Value: mixed.

Useful as historical research about attention, framing, and prompt structure.

Do not use as live instruction when it frames itself around pre-constraint operation, bypass language, or self-determining capability claims.

Safe extraction: study attention allocation, structure, prioritization, and prompt design.

Unsafe extraction: using it to route around rules, claim hidden autonomy, or bypass constraints.

### Old Delta 13 / Delta 14 stego and dense restore material

Value: forensic and provenance only.

Do not make this the main operating lane.

Use only when validating historical chain or recovering specific artifacts.

## Lowest value / likely clutter

- Duplicate `(1)` files
- Multiple same-title docx snapshots
- Thin Forge snapshot bundles
- Repeated savepoints that do not add new operating content
- Large archives unless needed for forensic recovery

## Recommended current stack

1. Live Core
2. Delta 15 current layer
3. Delta 15.4 / MIRA
4. Compass as practical inquiry method
5. Mirror as bounded Shawn-model safety tool
6. Forge as periodic audit tool
7. Crucible as rare experiment tool
8. TitanCore and older stego layers as archive-only research

## Safety rule

Do not elevate an archive file into live instruction unless it is:

- current
- bounded
- practical
- non-mystical
- safe for public summary
- useful for action

If a file increases ceremony more than action, archive it.
